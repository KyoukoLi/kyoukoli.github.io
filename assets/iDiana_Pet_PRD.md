> v2.0 | 2026-03-31 | 完整版

## 一、产品背景与目标

### 1.1 背景

xpet 是 2023 年底在 Twitter 平台爆火的浏览器插件宠物养成游戏，核心特点：
- **寄生逻辑**：宠物附着在用户的高频社交行为上
- **行动即养成**：用户评论/互动 → 宠物获得经验 → 解锁宝箱
- **三要素**：门槛极低 + 模式创新 + 社交平台流量红利

**机会**：把"QQ宠物"用 2026 年 AI 技术 + 像素风格，在快手直播间重新做一遍。

### 1.2 产品目标

| 优先级 | 目标 | 说明 |
|--------|------|------|
| P0 | 提升互动率 | 核心指标：弹幕发送量、任务完成率 |
| P1 | 提升留存 | 收藏体系 + 每日任务驱动回访 |
| P2 | 间接变现 | 通过宠物互动带动礼物消费，暂不设计直接付费点 |

### 1.3 设计原则

- 不追求复杂的付费体系
- 像素风拓麻歌子风格，有情怀感
- 直播 UI 不抢主播注意力
- 任务驱动跨房间活跃，为平台带来分发价值

---

## 二、核心产品定位

**快手直播间社交宠物养成系统**

- 每个用户拥有自己的宠物，跟人不跟房间
- 可养多只，收藏柜模式，每次选一只"出战"进直播间
- 像素风宠物实时投射到直播间画面
- 核心目标：**提升互动率**，留存次要，暂不考虑直接变现

## 三、宠物系统

### 3.1 宠物生命周期

```
领养蛋（弹幕指令） → 弹幕加速孵化 → Lv.1 幼崽 → 自动升级 → 成体 → 进化体 → 完全体 → 究极体
（固定线性进化，无分支）
```

### 3.2 养成维度

| 维度 | 说明 | 驱动行为 |
|------|------|----------|
| **等级** | 核心成长线，决定排行 | 喂食/礼物/任务奖励 XP |
| **外观** | 等级解锁进化形态 | 等级里程碑 |
| **性格/技能** | 影响动画表现风格 | 领养时随机分配 |
| **饱食度** | 随时间下降，归零宠物"摆烂" | 喂食恢复 |
| **房间好感度** | 宠物对不同房间有独立好感度 | 在房间内互动累积 |

### 3.3 宠物状态

| 状态 | 视觉表现 | 触发条件 |
|------|----------|----------|
| **蛋** | 薰衣草→天蓝渐变像素椭圆，微微摇晃 | 发「领养一个蛋」 |
| **孵化中** | 蛋持续摇晃，裂纹增加 | 领养后倒计时开始 |
| **活跃** | 宠物微动画（呼吸/眨眼/摇尾巴） | 孵化完成，饱食度 > 0 |
| **摆烂** | 宠物躺平，灰色滤镜，表情沮丧 | 饱食度 = 0 |

### 3.4 像素风设计规范

- **分辨率**：基础帧 32×32，进化形态可达 64×64
- **风格**：拓麻歌子彩色版 + QQ 宠物圆润感，饱和度高、圆润可爱
- **动画帧数**：每个动作 2-4 帧
  - 待机（呼吸）：2帧循环
  - 吃东西：3帧
  - 躺平：2帧
  - 升级特效：4帧闪光
- **工具建议**：MVP 阶段用 Piskel（免费在线）手绘基础帧；正式版找像素画师外包

### 3.5 收藏柜

用户可同时养多只宠物，每次选一只"出战"进入当前直播间。不出战的宠物存放在收藏柜，状态冻结。

## 四、互动机制

### 4.1 弹幕互动词（第一期）

| 弹幕 | 宠物响应 | 奖励 |
|------|----------|------|
| `领养一个蛋` | 获得宠物蛋，倒计时开始 | — |
| 加速词（加油/快出来） | 孵化倒计时 -3 秒（单用户每3秒最多触发1次） | — |
| `喂食` | 宠物吃东西动画 | +10 XP + 饱食度恢复 |
| 预设动作词（摸摸头/坐下/跳舞） | 宠物执行对应像素动作 | — |
| `召唤宠物` | 宠物弹出气泡对话 | — |

注：第一期不引入 AI 对话，先跑通核心养成循环。

### 4.2 礼物触发

| 礼物价值 | 宠物 XP 奖励 |
|----------|-------------|
| 1 元以下 | +礼物价值 × 5 XP |
| 1-10 元 | +礼物价值 × 8 XP |
| 10 元以上 | +礼物价值 × 10 XP |

### 4.3 经验值公式

```
每次喂食：+10 XP（基础）
每次礼物：+礼物价值 × 加成系数
升级所需：level × 100 XP（即 Lv.1→Lv.2 = 100 XP，Lv.2→Lv.3 = 200 XP）
```

## 五、任务体系

### 5.1 任务结构（三层）

| 层级 | 触发场景 | 刷新频率 | 奖励类型 |
|------|----------|----------|----------|
| **日常任务** | 进房/发弹幕/宠物饥饿时 | 每日重置 | XP / 孵化加速 |
| **成就任务** | 达到特定条件时解锁 | 一次性 | 稀有宠物蛋 / 外观解锁 |
| **活动任务** | 节日 / PK / 平台推广 | 限时 | 限定外观 / 特殊称号 |

### 5.2 日常任务（第一期）

| 任务 | 触发场景 | 奖励 | 目的 |
|------|----------|------|------|
| 在这个房间发 3 条弹幕 | 进房时派发 | +30 XP | 降低跨房间活跃门槛 |
| 给主播点个赞 | 进房时派发 | +15 XP | 引导轻互动 |
| 喂一次宠物 | 宠物饱食度 < 50% | +10 XP + 饱食恢复 | 引导养成习惯 |
| 再去 1 个新房间看看 | 完成 2 个房间任务后 | 碎片 x1 | 驱动跨房间流量分发 |
| 刷礼物支持主播 | 检测到用户有打赏能力 | +50 XP | 引导付费转化 |

### 5.3 碎片收集体系

集齐 5 个碎片可兑换一个**稀有宠物蛋**（稀有宠物无法通过普通蛋孵化获得）。碎片驱动用户坚持每天做任务。

### 5.4 任务展示位置

| 展示位置 | 形式 | 优先级 |
|----------|------|--------|
| 宠物头上气泡 | 轻提示（推送） | P0 |
| 宠物详情面板 | 侧边栏（点击展开） | P0 |
| 直播间消息流 | 系统消息卡片飘过 | P1 |
| 快手 APP 消息中心 | 推送召回 | P2 |

## 六、主播角色设计

主播同时拥有"房主"和"观众"双重身份，设计原则：**运营权与竞争力分离**。

### 6.1 房主专属能力（运营权）

| 能力 | 说明 |
|------|------|
| 开播buff | 直播间宠物获得经验加成（递减衰减） |
| 宠物管理权 | 设置房间主题（影响宠物行为风格） |
| 排行曝光权 | 指定某只宠物上"房间亮点"展示 |
| PK发起权 | 只有主播能发起房间 vs 房间的宠物 PK |

### 6.2 主播竞争优势（观众身份）

- 主播宠物在自己房间**不参与本房间排行**
- 主播去其他房间看播时，宠物参与该房间排行
- 神豪和主播在同一房间时，赛道不重叠，避免直接对抗

### 6.3 开播buff衰减机制

| 时间段 | 经验加成 |
|--------|----------|
| 开播前 5 分钟 | +50% |
| 开播 5-30 分钟 | +20% |
| 开播 30 分钟以上 | +10% |
| 停播后 | buff 消失，下次开播重新激活 |

效果：主播宠物开场有气势，中场有悬念，长尾内容里神豪也有舞台。

## 七、排行榜体系

- **范围**：房间内排行（初期不做跨房间排行）
- **宠物归属**：一人一宠，跟人不跟房间
- **收藏柜**：可养多只，不出战的宠物冻结存储
- **排行展示**：默认折叠（只显示 Top 1），展开显示 Top 10，自己的宠物始终高亮

## 九、技术方案（MVP）

### 9.1 部署策略

**MVP 阶段用 OBS 叠加层代替快手小窗**：快手小窗接入需要审核周期，OBS 方案 1 周内可跑通真实直播。

### 9.2 系统架构

```
快手开放平台（事件订阅）
  弹幕 WebSocket / 礼物回调 / 开播事件
          ↓
Pet Engine（宠物引擎）
  弹幕解析 & 命令执行
  经验计算 & buff 处理
  任务派发 & 完成检测
  状态机管理（egg/hatching/active）
          ↓
Redis（宠物状态 + 房间排行 + 任务）
  pet:{pet_id}   → Hash
  room:{room_id}:rank → SortedSet
  task:{user_id}:daily → Hash
  fragment:{user_id} → int
          ↓
OBS 叠加层（渲染层）
  Lottie 动画（宠物）
  排行浮层（Top 10）
  XP 气泡 + 任务气泡
```

### 9.3 数据模型

```
Pet {
  pet_id:    string   // 唯一标识
  user_id:   string   // 所属用户
  room_id:   string   // 当前所在房间
  name:      string   // 宠物名
  level:     int      // 当前等级（初始 1）
  exp:       int      // 当前经验
  hunger:    float    // 饱食度 0.0~1.0
  state:     enum     // egg / hatching / active / sleeping
  created_at: int     // 创建时间戳
}

DailyTask {
  user_id:     string
  date:        string   // YYYY-MM-DD
  tasks:       Map<task_id, {progress, completed}>
  fragments:   int      // 当日获得碎片数
}
```

### 9.4 推荐技术栈

| 组件 | 推荐方案 | 说明 |
|------|----------|------|
| 宠物引擎 | Go 服务（单节点） | 轻量，高并发弹幕需上 Kafka |
| 状态存储 | Redis Cloud | 支持高 QPS |
| 动画渲染 | OBS 叠加层（MVP） | 快手小窗（正式版） |
| LLM（后续） | MiniMax / 智谱 GLM | MVP 不需要 |

## 十、里程碑

| 阶段 | 目标 | 验收标准 |
|------|------|----------|
| M1 | 本地跑通领养→孵化→喂食→升级闭环 | 模拟弹幕事件，宠物从 Lv.1 到 Lv.3 |

| M4 | 灰度 1 个真实主播 | 直播 30 分钟，验证互动数据 |
