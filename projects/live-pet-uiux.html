直播虚拟宠物 MVP — UI/UX 设计方案

一、设计原则

直播间 UI 有且只有一个核心原则：不抢主播的注意力。

宠物是直播间氛围的"背景音"，不是主角。用户在欣赏主播的间隙，瞥一眼宠物觉得有趣，就够了。

  • 轻量化：UI 占屏幕面积不超过 15%，不遮挡主播画面
  • 即时反馈：用户发弹幕后 1 秒内有视觉反馈
  • 存在感强但不过度：宠物有持续微动画（呼吸/摇晃），但静止时不占视觉焦点
  • 颜色克制：主体颜色不超过 3 种，避免和主播画面撞色

2.2 宠物状态设计

状态视觉表现触发条件

2.3 宠物动画层级

事件动画里程碑触发等级提升/进化时升级金光、进化全屏特效

三、关键交互设计

3.1 领养蛋流程

✅ 用户视角
  1. 发弹幕「领养一个蛋」
  2. 屏幕右下角出现一颗摇晃的紫色蛋
  3. 看到提示「快来帮我加速孵化吧~」
  4. 继续发弹幕，看到倒计时减少
  5. 倒计时归零，蛋裂开，宠物出现
🔧 技术视角
  1. 弹幕解析 → 识别「领养」关键词
  2. Redis 创建 pet:{id}（state=egg）
  3. WebSocket 推送蛋 Lottie 动画
  4. 监听所有弹幕 → 倒计时 -3s
  5. 倒计时归零 → 推送孵化动画

3.2 喂食流程

✅ 用户视角
  1. 发弹幕「喂食」
  2. 宠物做出吃东西动画（0.5s）
  3. 弹出金色「+10 XP」文字
  4. 若升级，弹出「Lv.X ↑」并有金光特效
  5. 饱食度条短暂高亮
⚠️ 边界情况
  1. 饱食度已满 → 仍显示「吃撑了~」，无 XP
  2. 宠物在蛋中 → 显示「还没孵化呢~」
  3. 宠物已躺平 → 喂食后恢复，弹出「活过来了！」

七、技术实现建议

  • 动画引擎：Lottie（JSON 驱动，设计师出图，程序加载）
  • 宠物渲染：用 Spine 骨骼动画做交互动作，Lottie 做微动画
  • 渲染容器：快手小窗（需审核）或 OBS 叠加层（临时方案）
  • 排行榜更新：WebSocket 推送，渲染层每 5s 轮询兜底
  • 断线重连：宠物状态从 Redis 恢复，不丢失数据
