一、产品概述

1.1 产品定位

一款极简的文字生存与资源管理模拟器（对标《鳌太模拟器》）。

核心题材： 扮演极端狂热的"私生饭"，体验追踪偶像林屿森过程中的心理扭曲、窥探欲与道德拉扯。

情感基调： 心理惊悚、压抑、高试错成本带来的焦虑感 + 黑色幽默讽刺。

终极目标： 突破重重阻碍，拍到一张足以毁掉偶像职业生涯的绝密照片。

1.2 平台与形式

项目说明
------------
平台H5 网页（响应式，移动端优先）
部署地址https://idiana.wiki/projects/private-fan-simulator/
技术栈原生 HTML5/CSS3/JS + LLM 增强叙事
无需安装直接浏览器打开即可游玩

1.3 玩家背景

玩家扮演一个中产家庭出身的年轻人，辞去实习工作，谎称在北京工作，实际上追踪林屿森。父母以为子女在工作，实际上副卡已经刷爆。

四、叙事引擎

4.1 三段式叙事节奏

Phase 1 — RESULT（结果叙述） 玩家做出选择后，生成 80-120 字的结果叙述。风格：冷峻、克制、有感官细节。结果简短有力，像钉子一样扎进去。

Phase 2 — BUILDUP（事件铺垫） 自动生成 100-150 字的叙事铺垫。描写玩家当前处境（时间、地点、氛围、心理状态）。末尾要有悬念钩子，节奏慢下来。

Phase 3 — OPTIONS（选项呈现） 铺垫后呈现 2-3 个选项。每个选项要有明确的性格倾向（冒险/保守/极端），让玩家感受到每个选择代表不同的人生路径。

4.2 叙事风格

- 第一人称视角 - 冷峻、压抑、黑色幽默 - 注重感官细节：声音、气味、温度、触感 - 结果叙述：短促有力 - 铺垫叙述：制造悬念 - 禁止：暴力/色情描写

4.3 LLM Prompt 设计要点

系统 Prompt 包含： - 当前四维属性数值 - 追踪进度和天数 - 偶像信息（林屿森，顶流男演员） - 当前游戏阶段（萌芽/沉没/深渊/终局） - 三段式节奏说明 - 氛围参考表 - 严格 JSON 输出格式

六、技术架构

6.1 系统架构

前端 (H5)
  ↓ JSON 交互
LLM API (api.js) ←→ 本地引擎 (game.js)
  ↓                  ↓
OpenClaw/云LLM     事件库/结局库/偶像池

6.2 部署方案

- CSP 问题： GitHub Pages CSP default-src 'none' 阻止外部 JS - 解决方案： 所有代码内联到一个 HTML 文件（约90KB） - 构建脚本： scripts/bundle-sasaeng.py（Python） - 部署流程： 构建 → commit → push → CDN 传播（~2分钟）

6.3 数据文件

文件内容
------------
data/celebrities.js偶像配置（林屿森，单偶像）
data/events.js事件库（28个事件定义）
data/endings.json结局库（5坏3中1好+3终局+深渊）
data/config.js游戏参数配置（初始值/消耗/阈值）
prompts/game-agent.mdLLM Agent Prompt 模板
js/game.js本地游戏引擎（状态机/事件选择/结算）
js/api.jsLLM API 集成（三段式叙事/降级策略）
css/style.css暗色调 UI（心理惊悚风格）

6.4 API 降级策略

尝试调用 LLM API
    ↓ 失败
生成 Result 叙述（本地模板）
    ↓
生成 Buildup 叙述（本地事件描述）
    ↓
渲染选项（本地事件选项）
