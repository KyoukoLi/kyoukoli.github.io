# 私生模拟器 · 产品需求文档 (PRD)

> 版本：v1.0 | 最后更新：2026-04-02 | 状态：**MVP 开发中**

## 一、产品概述

### 1.1 产品定位

一款极简的文字生存与资源管理模拟器（对标《鳌太模拟器》）。

**核心题材：** 扮演极端狂热的"私生饭"，体验追踪偶像林屿森过程中的心理扭曲、窥探欲与道德拉扯。

**情感基调：** 心理惊悚、压抑、高试错成本带来的焦虑感 + 黑色幽默讽刺。

**终极目标：** 突破重重阻碍，拍到一张足以毁掉偶像职业生涯的绝密照片。

### 1.2 平台与形式

| 项目 | 说明 |
|------|------|
| 平台 | H5 网页（响应式，移动端优先）|
| 部署地址 | https://idiana.wiki/projects/private-fan-simulator/ |
| 技术栈 | 原生 HTML5/CSS3/JS + LLM 增强叙事 |
| 无需安装 | 直接浏览器打开即可游玩 |

### 1.3 玩家背景

玩家扮演一个**中产家庭出身**的年轻人，辞去实习工作，谎称在北京工作，实际上追踪林屿森。父母以为子女在工作，实际上副卡已经刷爆。

## 二、核心数值系统

### 2.1 四维属性

| 属性 | 初始值 | 归零后果 | 设计意图 |
|------|--------|---------|---------|
| 体力 (Stamina) | 100 | 猝死街头 Game Over | 时间压力 |
| 资金 (Money) | ¥5000 | 弹尽粮绝 Game Over | 核心约束 |
| 隐蔽度 (Stealth) | 100 | 社会性死亡 Game Over | 风险暴露 |
| 理智 (Sanity) | 100 | 锒铛入狱 Game Over | 心理防线 |

### 2.2 追踪进度

| 阶段 | 进度 | 氛围 | 机制 |
|------|------|------|------|
| 萌芽期 | 0-30% | 紧张但可控 | 可撤回，理智压力小 |
| 沉没期 | 30-70% | 焦虑上升 | 沉没成本增大，难以回头 |
| 深渊期 | 70-90% | 压力极大 | 理智上限开始衰减 |
| 终局 | 90-100% | 极度紧张 | 获得绝密照片，做出终局选择 |

### 2.3 每日结算

每回合（一天）的基础消耗：

- 体力：-5
- 资金：-¥50（基本食宿）
- 理智：-3（精神压力）
- 隐蔽度：±随机波动（-5 到 +5）

**租对面楼特殊状态：** 日常消耗打 8 折，每日自动获得进度 +3。

## 四、叙事引擎

### 4.1 三段式叙事节奏

**Phase 1 — RESULT（结果叙述）**
玩家做出选择后，生成 80-120 字的结果叙述。风格：冷峻、克制、有感官细节。结果简短有力，像钉子一样扎进去。

**Phase 2 — BUILDUP（事件铺垫）**
自动生成 100-150 字的叙事铺垫。描写玩家当前处境（时间、地点、氛围、心理状态）。末尾要有悬念钩子，节奏慢下来。

**Phase 3 — OPTIONS（选项呈现）**
铺垫后呈现 2-3 个选项。每个选项要有明确的性格倾向（冒险/保守/极端），让玩家感受到每个选择代表不同的人生路径。

### 4.2 叙事风格

- 第一人称视角
- 冷峻、压抑、黑色幽默
- 注重感官细节：声音、气味、温度、触感
- 结果叙述：短促有力
- 铺垫叙述：制造悬念
- 禁止：暴力/色情描写

### 4.3 LLM Prompt 设计要点

系统 Prompt 包含：
- 当前四维属性数值
- 追踪进度和天数
- 偶像信息（林屿森，顶流男演员）
- 当前游戏阶段（萌芽/沉没/深渊/终局）
- 三段式节奏说明
- 氛围参考表
- 严格 JSON 输出格式

## 五、结局系统

### 5.1 坏结局（Game Over，5个）

| 结局 | 触发条件 | 核心信息 |
|------|---------|---------|
| 弹尽粮绝 | 资金 ≤ 0 | 副卡刷爆，快捷酒店三天没吃饭 |
| 猝死街头 | 体力 ≤ 0 | 倒在机场到达大厅 |
| 社会性死亡 | 隐蔽度 ≤ 0 | 热搜第一，#私生饭跟踪林屿森# |
| 锒铛入狱 | 理智 ≤ 0 | 水果刀，精神鉴定，边缘型人格 |
| 偶像退圈 | 90天进度<50% | 九十个日夜，全部归零 |

### 5.2 中结局（3个）

| 结局 | 触发条件 | 核心信息 |
|------|---------|---------|
| 律师函警告 | 进度≥60%且隐蔽≤20% | 工作室律师函，永远进黑名单 |
| 职业代拍 | 进度≥50%且资金≤1000 | 从私生饭到职业代拍 |
| 及时止损 | 进度≥30%且理智≥50%且资源耗尽 | 回程火车上哭了很久 |

### 5.3 好结局（1个）

| 结局 | 触发条件 | 核心信息 |
|------|---------|---------|
| 神之私信 | 进度≥80%且理智≥60%且隐蔽≥40% | 偶像私信："请你放过自己" |

### 5.4 终局分支（进度100%时）

玩家获得绝密照片后，三选一：

| 选项 | 结局名 | 效果 |
|------|--------|------|
| 曝光 | 玉石俱焚 | 偶像退圈，你也面临法律制裁 |
| 变现 | 非法变现 | 卖照片赚60万，回家，但失去了相信的勇气 |
| 保留 | 神之视角 | 深渊模式：守着秘密的幽灵，理智上限持续衰减 |

### 5.5 深渊模式（神之视角后续）

进入条件：终局选择"保留"

- 每日理智上限 -2
- 解锁特殊场景：被害妄想、秘密泄露
- 特殊结局：电脑被黑，秘密全网曝光

## 六、技术架构

### 6.1 系统架构

```
前端 (H5)
  ↓ JSON 交互
LLM API (api.js) ←→ 本地引擎 (game.js)
  ↓                  ↓
OpenClaw/云LLM     事件库/结局库/偶像池
```

### 6.2 部署方案

- **CSP 问题：** GitHub Pages CSP `default-src 'none'` 阻止外部 JS
- **解决方案：** 所有代码内联到一个 HTML 文件（约90KB）
- **构建脚本：** `scripts/bundle-sasaeng.py`（Python）
- **部署流程：** 构建 → commit → push → CDN 传播（~2分钟）

### 6.3 数据文件

| 文件 | 内容 |
|------|------|
| `data/celebrities.js` | 偶像配置（林屿森，单偶像）|
| `data/events.js` | 事件库（28个事件定义）|
| `data/endings.json` | 结局库（5坏3中1好+3终局+深渊）|
| `data/config.js` | 游戏参数配置（初始值/消耗/阈值）|
| `prompts/game-agent.md` | LLM Agent Prompt 模板 |
| `js/game.js` | 本地游戏引擎（状态机/事件选择/结算）|
| `js/api.js` | LLM API 集成（三段式叙事/降级策略）|
| `css/style.css` | 暗色调 UI（心理惊悚风格）|

### 6.4 API 降级策略

```
尝试调用 LLM API
    ↓ 失败
生成 Result 叙述（本地模板）
    ↓
生成 Buildup 叙述（本地事件描述）
    ↓
渲染选项（本地事件选项）
```
